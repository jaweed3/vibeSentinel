How It Works

1

Sample

LSM6DS3 reads 3-axis acceleration at 200Hz. 128 samples per window (640ms), 50% overlap between consecutive windows.

imu.read_accel(&mut buf); // 200Hz, I2C
2

Extract

Hann window → 128-pt RFFT → magnitude spectrum. Per-axis: RMS, Peak, Kurtosis, Crest Factor, FFT bins 0-3. Cross-axis: Axial/Radial ratio, Total RMS. → 26-dim feature vector.

let features = extract_features(&window); // 26 f32
3

Infer

Z-score normalization (clip ±3). Autoencoder forward pass: 20→10→4→10→20 with ReLU/Sigmoid. 524 parameters, ~2KB, <1ms on ESP32-S3.

let mse = model.forward(&normalized); // <1ms
4

Detect

MSE reconstruction error > ANOMALY_THRESHOLD? 3-sigma calibrated during training. Triggers GPIO LED alert (E010) + structured serial health report.

if mse > THRESHOLD { alert.set_high(); } // E010

See It In Action

Cargo test — 31 unit tests across feature extraction and model inference. All pass.

Training — 50 epochs on synthetic vibration data. Adam optimizer. Calibration report with 3-sigma threshold.

Golden Vectors — 5 pre-computed inputs verify x86 output matches Xtensa output within 1e-5 tolerance.

Export — Auto-generated weights.rs with weights, biases, threshold, mean, std, and golden vectors.
